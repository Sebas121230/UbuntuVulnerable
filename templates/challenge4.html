Blind Boolean SQL Injection

No hay mensajes de error. Solo una respuesta binaria: el usuario existe o no existe.
Nivel 1: La respuesta indica claramente si el usuario existe.
Nivel 2: Respuesta genérica — hay que inferir con condiciones booleanas.

1
BOOLEAN RESPONSE

La página dice claramente si el ID existe o no. Úsalo para extraer datos carácter a carácter.

ACCEDER
2
TIME-BASED

Respuesta siempre igual. Necesitas sqlmap o lógica de tiempo/inferencia para extraer datos.

ACCEDER
💡 Técnica: Blind Boolean

Payloads de ejemplo:
?id=1 AND 1=1-- → existe (TRUE)
?id=1 AND 1=2-- → no existe (FALSE)
?id=1 AND SUBSTR(username,1,1)='a'-- → extrae carácter a carácter
